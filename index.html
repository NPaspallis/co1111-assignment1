Sample Treasure Hunt app

This is a sample project for CO1111's assignment, aka the Treasure Hunt app. It describes the minimum content you need to have in your app.
As this is the 'index' page, it will be the landing point for everyone visiting your team's page.
Note some important requirements:
  • This is the landing page. Make sure it looks good and explains clearly the purpose of this page.
  • One of the main purposes of this landing page is to market your app. So use what you have learned in class to explain your advantages (make it wort-while for your visitors).
  • The recommendation is to keep all your marketing/video/info details in a single page but it's ok if you decide to split that to multiple pages.
  • Make an effort to use best practice approaches, splitting the content (HTML) and design properties (CSS) to separate files.
  • Your page and app should have a 'mobile-first' design, meaning it should certainly be functional and appropriate for mobile devices. But also test it on desktop screens. If you properly use RWD principles the page should automatically adjust as needed.
  • You are encouraged to not use any of the styling frameworks (such as Bootstrap or Material etc.). I know they are good, but it's better if you try try to build your own web-app from scratch for scratch, so you get the right experience.
  • Make sure to add a clearly visible link such as a button to allow opening your web-app page, as shown above.
  • Please keep all your JavaScript in a single file, e.g. named th-api.js. Follow the instructions in the handout for defining the right functions.
  • There is also a requirement to specify a test.html page that includes your testing scripts.
  • Please note this project is only a reference. You are not required to clone it or use it as the starting point (but feel free to do so if you wish). Your definite guide is the assignment handout itself though.
